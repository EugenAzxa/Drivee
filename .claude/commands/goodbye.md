The session is ending. Work through these steps in order.

STEP 1 — Review the work
Read index.html and compare it to where we started today.
List what was actually built or changed this session.
Be specific — name the features, not just "made progress".

STEP 2 — Check for issues
List anything broken, incomplete, or worth flagging for next session.
If nothing is broken, say so clearly — that is worth noting too.
Also read PRD.md and check if anything built today conflicts with, extends, or is missing from it.
If so, note it — the PRD may need updating next session.

STEP 3 — Write the session summary
Append a new entry to SESSION-NOTES.md with today's date and session number.
Include: what was built (3 to 5 points), any issues to carry forward, homework set, plan for next session.

STEP 4 — Set homework
Suggest one task the student can do alone before next session.
It should take 15 to 20 minutes maximum.
It should reinforce something from today, not introduce anything new.
Tell them exactly what to do and what to search for if they get stuck.

STEP 5 — Close out
End with one sentence of encouragement.
Then say: "Type /hello at the start of your next session to pick up where we left off."

Use plain text throughout. No markdown symbols. No hashtags or asterisks.
